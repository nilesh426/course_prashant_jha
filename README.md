What is this?
This repo contains Python scripts grouped by lesson days. Each day builds on the previous one, covering everything from basic syntax to advanced topics like recursion and data structures. The examples are simple, with comments to explain the code. It's great for:

New learners starting with Python.
Students practicing for exams or projects.
Developers reviewing concepts like OOP or file handling.
Anyone who wants to code small programs and see results quickly.
The focus is on understanding through practice, not just theory.

What you need
Before starting, make sure you have:

Python 3.8 or newer: Download from python.org if needed. Check your version with python --version in the terminal.
Command line access: Use Command Prompt (Windows), Terminal (Mac), or any shell to run scripts.
Text editor or IDE: Something like VS Code, PyCharm, or even Notepad to view and edit files. VS Code is free and works well with Python.
Basic computer skills: Knowing how to open folders and type commands is helpful.
No advanced setup required—just install Python and you're ready!

What each day covers
Here's a breakdown of each day's topics, with key concepts and example files:

DAY1 🏗️: Foundations of Python
Conditionals: if, elif, else for decision-making.
Operators: Comparison, logical, membership, and identity checks.
Basics: Arithmetic, type casting, swapping variables.
Examples: greatest.py (find largest number), simpleinterest.py (calculate interest), reverse.py (reverse a string).
Files: conditional1.py to conditional7.py, marks.py, etc.

DAY2 🔄: Working with data and loops
Lists: Creating, accessing, and modifying lists.
Loops: for and while for repetition.
Strings: Slicing, functions, and manipulation.
Examples: tables.py (multiplication tables), string_fun.py (string operations), Tuple.py (tuple basics).
Files: List.py, looping.py, STRING_SLICING.PY, etc.

DAY3 📋: Functions and data storage
Functions: Defining, calling, and using arguments.
Dictionaries: Key-value pairs for data storage.
Loops: Nested loops for complex patterns.
Examples: function.py (basic functions), Dict.py (dictionary use), menu_driven.py (interactive menu).
Files: nestedforloop.py, positional_arg.py, strip.py.

DAY4 ⚠️: Handling errors and files
Exceptions: try, except, finally for error management.
Files: Reading and writing text files.
Built-ins: Using Python's predefined functions.
Examples: Exceptionhandling.py (error examples), filehandling.py (file I/O), predefined.py (utility functions).
Files: tasks.py.

DAY5 🏛️: Object-oriented programming basics
Classes: Defining classes and creating objects.
Methods: Instance methods and constructors.
OOP: Inheritance, polymorphism, and overriding.
Examples: Inheritance.py (class hierarchies), Polymorphism.py (method overriding), Method.py (class methods).
Files: objectorientedprogramming.py, prac.py, task.py.

DAY6 🔒: Advanced OOP and data structures
Abstraction: Hiding details with abstract classes.
Encapsulation: Protecting data in classes.
Stacks: Implementing stacks with size limits.
Examples: Abstraction.py, Encapsulation.py, stackwithsizelimit.py, TowerofHanoi.py (recursive puzzle).
Files: prac.py, DSA folder with stack examples.

DAY7 ⏱️: Efficiency and linked structures
Time complexity: Understanding code speed.
Linked lists: Building and managing linked lists.
Queues: Queue implementation and operations.
Examples: linked_list.py, queue.py, menu_driven_linked_list.py (interactive list management).
Files: time_complexity.txt, prac.py.

DAY8 🔍: Text processing with regex
Regex: Using re module for patterns.
Functions: match, search, findall, sub, etc.
Validation: Checking emails, strings, etc.
Examples: match_func.py, findall_function.py, validation.py (pattern matching).
Files: All in Regular_Expression/ folder.

DAY9 🔄: Recursion
Recursion: Functions calling themselves.
Problems: Factorial, Fibonacci, palindromes.
Comparison: Recursion vs. iteration.
Examples: recursion.py (with factorial and power functions), comparisons in code.
Files: recursion.py.

DAY10 🧩: Practice problems (nested loops)
Problem solving: Counting pairs with a target difference using loops.
Examples: prac.py (counts how many pairs have prices[j] - prices[i] == k).
Files: prac.py.

How to run the code
Open your terminal or command prompt.
Navigate to the day's folder, e.g., cd DAY1.
Run a script: python conditional1.py.
Watch the output! Try editing the code (e.g., change numbers) and run again to see changes.
For interactive scripts, follow on-screen prompts.
Example command:

cd course_prashant_jha/DAY2
python List.py
If you get errors, check Python is installed and the file path is correct.

Learning path
Follow the days in order for the best experience:

Start with DAY1 for basics.
Move to DAY2-3 for loops and functions.
DAY4 for real-world skills like files.
DAY5-6 for OOP.
DAY7 for data structures.
DAY8-9 for advanced topics.
If a topic is tough, revisit earlier days or search online for help. Practice daily!
