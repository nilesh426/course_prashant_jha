## 🌟 About This Repository

This repository is a step-by-step Python learning journey organized by days. Each day focuses on a specific set of topics so you can learn gradually and practice with small, easy-to-understand programs.

It is a good fit for:

- 👨‍🎓 Beginners starting Python from scratch
- 📝 Students practicing for assignments, exams, or mini-projects
- 💻 Developers revising Python fundamentals
- 🚀 Anyone who wants to learn by writing and running code

> The main goal of this repository is learning through practice, not just reading theory.

## 🛠️ Requirements

Before you begin, make sure you have:

- **Python 3.8 or newer**
  Install it from [python.org](https://www.python.org/).

- **Terminal / Command Line access**
  Use Command Prompt, PowerShell, Terminal, or any shell you are comfortable with.

- **A code editor**
  VS Code, PyCharm, or any text editor that supports Python files.

- **Basic computer knowledge**
  Knowing how to open folders and run commands is enough to get started.

Check your Python version with:

```bash
python --version
```

## ▶️ How to Run the Code

1. Open your terminal or command prompt.
2. Move into the folder for the day you want to practice.
3. Run the Python file using `python filename.py`.

### Example

```bash
cd course_prashant_jha/DAY2
python List.py
```

For interactive scripts, follow the prompts shown on the screen.

If something does not work, check these first:

- ✅ Python is installed properly
- ✅ You are in the correct folder
- ✅ The file name is typed correctly

## 🛤️ Suggested Learning Path

To get the best experience, follow the repository in order:

1. Start with `DAY1` to build your Python basics.
2. Continue to `DAY2` and `DAY3` for loops, strings, functions, and dictionaries.
3. Use `DAY4` to learn useful real-world concepts like files and exceptions.
4. Move to `DAY5` and `DAY6` for object-oriented programming and stack-based thinking.
5. Practice `DAY7` for linked structures and efficiency.
6. Explore `DAY8`, `DAY9`, and `DAY10` for regex, recursion, trees, and graphs.

> If a topic feels difficult, go back, rerun examples, and modify the code yourself. That is the fastest way to improve.
